@ModelAttribute("regions")
public Map<String,String> regions() {
        Map<String, String> regions = new LinkedHashMap<>();
        regions.put("SEOUL", "서울");
        regions.put("BUSAN", "부산");
        regions.put("JEJU", "제주");
        return regions;
}
-> 만약에 어떤 컨트롤러에서 각 메서드마다 Model 객체를 통해 뷰에 넘겨주는 데이터가 공통적인 게 있다면, 그걸 일일이 각 메서드마다 중복해서 넣어 줘야 한다. 
이걸 방지하기 위하여 위처럼 @ModelAttribute를 사용하면 해당 컨트롤러 실행 시 모든 메서드의 Model에 "regions" 데이터가 자동으로 포함되게 되면서 코드 중복을 막아 준다. 

그러나 이 방식은 그렇게 추천하지 않는다.
왜냐하면 컨트롤러가 호출될 떄마다 실행되기 때문에 메모리 낭비도 있고, 굳이 호출될 때마다 실행될 필요는 전혀 없다.
그래서 자주 사용되는 Model 값들은 클래스를 만들어서 필요시마다 static으로 호출하는 것이 더 효율적이다.
